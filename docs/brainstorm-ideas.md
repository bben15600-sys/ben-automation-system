# רעיונות לפיתוח — ייעול החיים עם אוטומציה

> תאריך: 2026-04-15
> מבוסס על התשתית הקיימת: Claude Code + Notion + Telegram + GitHub Actions

---

## רמה 1 — שדרוגים למה שכבר קיים

### 1. סקירה שבועית אוטומטית (Weekly Review Agent)
**מה:** בסוף כל שבוע — סיכום אוטומטי בטלגרם: כמה הוצאת, כמה הרווחת מ-VR, איפה אתה ביחס ליעד הרכב (40K), האם השקעת ב-IBKR החודש.
**למה:** במקום לבדוק ידנית, הכל מגיע בהודעה אחת ביום שישי.
**מורכבות:** נמוכה — קריאה מ-DB קיימים + חישוב + שליחה בטלגרם.
**DB נדרשים:** Budget, VR Events, Investments (כולם קיימים)

### 2. בריף בוקר אישי (Morning Brief Agent)
**מה:** כל בוקר ב-07:00 הודעה בטלגרם — מה יש היום בלוח, מה קרה בשוק אתמול, תזכורות לדדליינים של פרויקטי וידאו, מזג אוויר.
**למה:** הכל במקום אחד, מתחיל את היום ממוקד.
**מורכבות:** בינונית — צריך GitHub Action חדש עם Cloud Scheduler יומי.
**DB נדרשים:** Weekly Schedule, Daily Brief, Video Projects (כולם קיימים)

### 3. אנליטיקות VR חכמות
**מה:** ניתוח עסקי: אילו ימים הכי רווחיים, ממוצע לאירוע, טרנד חודשי, תחזית הכנסה לחודש הבא.
**למה:** עוזר לקבל החלטות עסקיות — מתי לקחת אירועים, האם להעלות מחיר.
**מורכבות:** נמוכה — קריאה מ-VR Events DB + חישובים סטטיסטיים.
**DB נדרשים:** VR Events (קיים)

## רמה 2 — סוכנים חדשים

### 4. מעקב הרגלים (Habit Tracker Agent)
**מה:** DB חדש בנושן — כל יום מסמן: התאמנת? שתית מים? למדת קורס? קראת? ישנת 7+? סיכום שבועי עם אחוזי עקביות.
**למה:** בונה שגרה בלי אפליקציה נפרדת. הכל בתוך המערכת.
**מורכבות:** בינונית — DB חדש + סוכן + אינטגרציה עם בריף בוקר.
**DB חדש:** Habits

### 5. מעקב קשרים (Relationship Tracker Agent)
**מה:** DB שעוקב מתי בפעם האחרונה ראית אנשים חשובים (אבא, סבא וסבתא, חברים). אם עברו X ימים — תזכורת בטלגרם.
**למה:** "לא ראית את סבא כבר 3 שבועות" — מונע מצב שהזמן עובר בלי לשים לב.
**מורכבות:** נמוכה — DB פשוט + סוכן שבודק תאריכים.
**DB חדש:** Relationships

### 6. ניהול פרילנס (Freelance Manager Agent)
**מה:** DB שעוקב: לקוח, פרויקט, מחיר מוסכם, סטטוס תשלום, תאריך חשבונית. התראה על תשלומים שלא התקבלו.
**למה:** סיכום חודשי של הכנסות פרילנס, מניעת "שכחתי לגבות".
**מורכבות:** בינונית — DB חדש + אינטגרציה עם Budget DB.
**DB חדש:** Freelance Projects

### 7. תכנון ארוחות (Meal Planner Agent)
**מה:** לפי סוג השבוע (בסיס/בית) — תפריט שבועי מותאם + רשימת קניות אוטומטית.
**למה:** חוסך "מה לאכול" כל יום, מפחית בזבוז אוכל וכסף.
**מורכבות:** בינונית — DB חדש + לוגיקה של מתכונים.
**DB חדש:** Meals / Recipes

## רמה 3 — אוטומציות חכמות

### 8. התראות תקציב בזמן אמת (Smart Budget Alerts)
**מה:** התראה בטלגרם כשעוברים 80% מתקציב בקטגוריה. "הוצאת 1,200 ₪ על אוכל בחוץ, נשארו 300 ₪."
**למה:** חוסך כסף בפועל — מודעות לפני שעוברים את הגבול.
**מורכבות:** נמוכה — בדיקה יומית של Budget DB מול תקרות שהוגדרו.
**DB נדרשים:** Budget (קיים) + הגדרת תקרות

### 9. התראת איזון תיק (Investment Rebalance Alert)
**מה:** הסוכן מזהה סטייה של 5%+ מהאלוקציה ושולח: "QQQ ירד ל-12%, היעד 15%. שווה לקנות עוד $X."
**למה:** ממש אקשן-אייטם — לא רק מידע, אלא המלצה קונקרטית.
**מורכבות:** נמוכה — שדרוג ל-Daily Brief Agent הקיים.
**DB נדרשים:** Investments, Daily Brief (קיימים)

### 10. יצירת חשבוניות אוטומטית (Auto-Invoice Generator)
**מה:** לאירועי VR ולפרויקטי וידאו — PDF אוטומטי מנתוני נושן.
**למה:** חוסך עבודה ידנית, מונע שכחה.
**מורכבות:** בינונית — צריך template + ספריית PDF.
**DB נדרשים:** VR Events, Video Projects (קיימים)

### 11. מעקב לימודים (Course Progress Tracker)
**מה:** באיזה קורס אתה, כמה שעות למדת, כמה נשאר, קצב צפוי לסיום. "בקצב הנוכחי תסיים בעוד 6 שבועות."
**למה:** מוטיבציה + תכנון ריאלי + שילוב עם לוח שבועי.
**מורכבות:** בינונית — DB חדש + אינטגרציה עם Weekly Scheduler.
**DB חדש:** Courses

## רמה 4 — חוצה-מערכת

### 12. שליטה מרחוק בטלגרם (Telegram Remote Control)
**מה:** שליטה מלאה מטלגרם: "תראה תקציב", "תוסיף הוצאה 50 אוכל", "מה יש מחר", "כמה הרווחתי מ-VR."
**למה:** הכל בצ'אט בלי לפתוח Notion או מחשב.
**מורכבות:** גבוהה — Telegram bot עם NLP + routing לסוכנים.
**סטטוס:** מתוכנן, עדיין לא מומש.

### 13. דשבורד אישי (Personal Dashboard)
**מה:** שדרוג אתר GitHub Pages — תקציב חודשי, התקדמות יעד רכב, הכנסות VR, ביצועי תיק, הרגלים.
**למה:** מבט אחד על הכל. לא רק לוח שבועי.
**מורכבות:** בינונית — הרחבת generate_site.py.
**DB נדרשים:** כל ה-DB הקיימים.

### 14. סנכרון Notion ↔ Google Calendar
**מה:** כל מה שנכנס ללוח בנושן — אוטומטית גם בגוגל קלנדר עם תזכורות. ולהיפך.
**למה:** תזכורות push אמיתיות בטלפון.
**מורכבות:** גבוהה — צריך Google Calendar API + סנכרון דו-כיווני.
**DB נדרשים:** Weekly Schedule (קיים)
